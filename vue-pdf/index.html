What is VueJS ?

VueJs is a progressive framework and it is used to build dynamic user interface and Single Page Applications. The core library is mainly focused on view-layer only. It is easily pick-up and intergrates with other libraries or existing projects.

  • VueJS let we extend HTML with HTML attributes called directives
  • VueJs directives offers functionality to HTML applications
  • VueJs provides built-in directives and user defined directives

Advantages of using VueJS

Small-in-size

The size of the framework is 18kb to 21kb and it takes no time for user download and use it.

Simple Integration

It is easily integrated with other libraries or existing projects

Flexibility

The flexibility also makes easy to understand for the developers of ReactJS, AngularJS and any other new JavaScript frameworks.

Easy to understand

One of the reason for the popularity of this framework is easy to understand. The user can easily add vuejs on his project because of the simple structure.

Virtual DOM

It uses virtual DOM similar to the other existing projects like ReactJS, AngularJS etc. Virtual DOM is a light weight in-memory tree representation of the original HTML DOM and it updates without effecting original HTML DOM.

Components

It is used to create reusable custom elements in Vuejs application

Two-way data binding

VueJS facilities Two-way communication because of "MVVM" architecture which make's it quite easy to handle HTML blocks.

Directives

Directives are instructions for vuejs to do things in a certain way. Built-in directives are v-if, v-else, v-for, v-text, v-html, v-repeat, v-on, v-bind, v-attr, v-show, v-style, v-class etc.,
Literal directives are v-transition, v-ref and v-el
Empty Directives are v-pre and v-cloak

v-text directive:

Instead of interpolation, we can use v-text directive and it performs same job as interpolation
or
v-text directive updates an element textContent. Internally, {{mustache}} are also compiled as v-text directive on a textNode

v-once directive:

It is used to avoid unwanted re-renders of an element. It treats the element is a static content after rendering it once for the first time. It improves the performance as it does not have to be rendered again.

v-html directive

v-html directive updates the element's inner html. Using v-html, with users supplied data can be dangerous. It is suggested that you use v-html when you are absolutely sure about data source or pipe it through custom filters that sanitizes untrusted HTML.

v-bind directive:

It binds one or more attributes or a component props to an elements. if that attribute is bonded to our data defined

Form Input Bindings

v-model directive:

It allows us to bind two-way data binding. We can bind Form input element for example makes it changes Vue data property when the user changes the content of the field.

Conditional Rendering

v-if directive

The directive v-if is used conditionally rendered a block. The block will only be rendered if the directive's expression returns truthy value

Conditional groups with v-if on a template eleement

Because v-if is a directive, it has to be attached to a single element. But what if we want to toggle with more than one element? In this case, we can use v-if on a template element which serves as a invisible wrapper. The final rendered result will not include template element.

v-else directive

We can use v-else directive to indicate 'else block' for "v-if" directive.
v-else directive must immediately follow v-if or a v-else-if elements, otherwise it will not be recognized.

v-for directive

We can use v-for directive to render list of items based on an array. The v-for directive requires special syntax like "item in items". Where as items is a source data array and item is an alias array element is iterated on
